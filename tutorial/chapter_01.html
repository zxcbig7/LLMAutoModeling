1你需要準備什麼

OptimFoundation 是一套用 C# 封裝商用求解器（CPLEX / Gurobi）的開發框架。開始前先備齊四樣東西：

項目說明怎麼確認
.NET 8 SDK編譯與執行環境（框架以 net8.0 為目標）dotnet --version ≥ 8.0
IDEVisual Studio 2022 / VS Code / Rider 擇一能開 .sln 或 .csproj
求解器IBM ILOG CPLEX（本教學主用）或 Gurobi有對應的 ILOG.*.dll / Gurobi DLL
OptimFoundation DLL框架本體（Core + Cplex）dlls\ 內五個檔齊全

3安裝最佳化求解器（CPLEX / Gurobi）

OptimFoundation 把「建模」和「求解器」拆開：你寫的模型程式碼透過 OptimFoundation.Cplex 或 OptimFoundation.Gurobi 兩種引擎之一去求解。本教學全程使用 CPLEX，但同一套建模 API 換成 Gurobi 引擎也能跑。

3.1 CPLEX

安裝 IBM ILOG CPLEX Optimization Studio。安裝後你會得到兩個關鍵 DLL：ILOG.Concert.dll（建模介面）與 ILOG.CPLEX.dll（求解器）。

授權版本的差別（很重要）

Community Edition 免費，但限制 1,000 個變數、1,000 條約束，超過會求解失敗或被截斷。學術版（IBM Academic Initiative，免費）與商業版則無此上限。本教學的工廠範例很小，Community Edition 也跑得動；但實務模型動輒上萬變數，務必確認你用的是完整授權。

3.2 Gurobi（替代方案）

若改用 Gurobi：安裝 Gurobi、取得授權檔 gurobi.lic，並設定環境變數讓求解器找得到它。學術授權免費。

怎麼知道授權有沒有過

把第 7 節的 Hello World 跑起來：若 status = Optimal 且求解器 log 沒有 size-limited / promotional 之類的警告，代表求解器與授權都就緒。授權問題通常會在 Solve() 階段丟例外（見第 8 節）。

6標準專案結構

OptimFoundation 的專案採「數學模型一個資料夾、程式一個資料夾」的對應式結構。每個資料夾對應數學模型的一個部分，方便把「建模」和「實作」一一對照：

開發兩階段原則

框架精神是 先建模、後實作：在 Model/ 把數學模型寫清楚（下一章開始講），程式只是把模型「機械式」翻成 code。這個結構讓每個 .cs 都能對回模型的某一條，往後三章我們會逐一填滿這些資料夾。

8疑難排解

① 找不到型別 OptEngine / CplexConfig

錯誤error CS0246: 找不到型別或命名空間名稱 'OptEngine'
原因csproj 的 HintPath 相對路徑層數算錯，或 dlls\ 少檔。
解法確認 Projects\專案\ 下用 ..\..\dlls\；五個 DLL 都在；using OptimFoundation.Cplex; 與 using OptimFoundation.Core; 都有加。

② 執行期找不到原生 CPLEX

錯誤DllNotFoundException: cplex<版本>.dll 或 Unable to load ILOG.CPLEX
原因ILOG.CPLEX.dll 是 .NET 包裝，底層還要呼叫 CPLEX 的原生 DLL；CPLEX 安裝目錄的 bin 不在系統 PATH。
解法把 CPLEX 安裝目錄下的 ...\cplex\bin\x64_win64 加進系統 PATH，重開終端機。

③ 變數/約束數量超過上限

症狀求解丟例外，或 log 出現 1000 variables / size-limited 警告。
原因用的是 CPLEX Community Edition，超過 1,000 變數 / 約束上限。
解法改用學術版或商業版授權；本教學範例很小不會踩到，但實務模型會。
